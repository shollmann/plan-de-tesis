Qué es el Open Source

  • Software Libre
  • Open Source
  • Software Privativo
El software, según las libertades de sus usuarios, puede clasificarse en Software Libre, Open Source y Software Privativo. En esta tesís se habla de Software Libre y Open Source como sinónimos ya que la manera de desarrollarlos es prácticamente la misma y sus diferencias son filosóficas.

Importancia del Open Source

  • Super computadoras
  • Dispositivos móviles
  • Aviones
  • La Bolsa de Valores
El software de código abierto está presente en todos lo ámbitos. Es dificil de imaginar un mundo sin este tipo de programas. Los dispotivos móviles, electrodomésticos, aviones, gobiernos, etc utilizan el software de código abierto para desempeñar sus funciones.

Ejemplos de Open Source

Firefox (y Opera) allanaron el camino para que otros browsers se liberen como open source y soporten estándares web. Incluso Explorer hoy en día hace mucho para soportar y promover los estándares web. El desarrollo de proyectos de código abierto está en auge. Herramientas como GitHub ayudan a que sea cada vez más común que desarrolladores creen proyectos de este tipo.

Problemática

  • Esfuerzo duplicado
  • Enorme cantidad de forks
  • Poca de visibilidad
  • Falta de voluntarios
Si bien el software de código abierto busca reutilizar el código, es decir, no reinventar la rueda, en la actualidad el esfuerzo duplicado es enorme. Los proyectos de código abierto no tienen buena difusión, no captan voluntarios, tienen poca colaboración y escasa visibilidad.

Problemática

Existe una enorme cantidad de proyectos que buscan el mismo objetivo y que utilizan la misma tecnología para alcanzarlo. De esta manera, se desperdicia un montón de esfuerzo.

Orígenes del problema

  • Malas prácticas de programación
  • Poca comunicación en el proyecto
  • Escasa documentación
  • Carencia de organización
  • Otros
Existe una gran diversidad de detonadores que ayudan a la creación de proyectos nuevos o forks de proyectos existentes. La falta de buenas prácticas de programación, la escasa documentación o carencia de organización, son algunos de los motivos que auyentan a los voluntarios potenciales. Otros orígenes del problema son: la falta de mantenimiento de los proyectos, la carencia de herramientas, la falta de apertura, etc.

Malas prácticas de programación

Una buena práctica fundamental de progración es la utilización de nombres de variables que describan al dominio del problema. El algoritmo puede ser sumamente sencillo pero si las variables no describen el dominio será casi imposible entender qué es lo que hace.

Malas prácticas de programación

Al mejorar los nombres de las variables y métodos, es más fácil de entender el algoritmo.

Malas prácticas de programación

El código debe ser documentación en si mismo. Se debe poder leer como texto y describir en detalle qué es lo que hace.

Solución / Objetivo

Crear un marco de trabajo para la creación de proyectos de código abierto y que garantice su escalabilidad mundial.

Esta tesis pretende crear un marco de trabajo para la creación de proyectos de código abierto. De esta manera se busca redefinir el concepto de código abierto y suministrar una serie de buenas prácticas y guías para la creación de este tipo de proyectos.

Objetivos específicos

  • Redefinir el concepto de código abierto
  • Identificar las mejores prácticas para el desarrollo de sistemas de código abierto
  • Formular una fuente de contenidos sobre el desarrollo de proyectos de código abierto
  • Crear un marco de trabajo para proyectos de código abierto
Los objetivos específicos permiten alcanzar el objetivo principal de la tesis.

Objetivos específicos

  • Describir y analizar dos casos de proyectos de código abierto exitosos
  • Implementar el marco de trabajo creado en dos casos de prueba
  • Recopilar información sobre la implementación en los casos de prueba
  • Analizar y sacar conclusiones sobre la información recopilada
  • Realizar un resumen del marco de trabajo
Los objetivos específicos permiten alcanzar el objetivo principal de la tesis.
